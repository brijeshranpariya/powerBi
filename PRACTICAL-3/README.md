# Data Leverage – Power BI Practical 3

## Project Overview

In this project, I worked with multiple datasets related to sales, customers, products, regions, dates, and returns. Using Power BI, I performed data preparation, relationship modeling, and report creation to simulate a real-world business intelligence workflow.

---

# Dataset Used

The project was built using the following datasets:

- Date_Dim
- Product_Dim
- Customer_Dim
- Region_Dim
- Sales_Fact
- Returns_Fact

These datasets together formed a star schema model where dimension tables were connected with fact tables for analytical reporting.

---

# Work Performed in Power BI

## 1. Data Import and Preparation

- Imported multiple datasets into Power BI.
- Verified and corrected column data types.
- Cleaned and organized the datasets for reporting.
- Renamed columns wherever required for better readability.
- Removed unnecessary fields and ensured data consistency.

---

## 2. Data Modeling

- Created relationships between fact and dimension tables.
- Implemented a star schema approach for efficient reporting.
- Connected sales and returns data with supporting dimensions like:
  - Customer
  - Product
  - Region
  - Date

- Managed relationship directions and resolved modeling issues during implementation.

---

## 3. DAX Calculations

In this practical, DAX measures were created to perform business calculations and analytical comparisons.

Some of the analytical calculations included:

- Total Sales
- Total Returns
- Monthly Sales Analysis
- Current vs Previous Month Comparison
- Time Intelligence Calculations
- Aggregated KPI calculations

This helped in understanding how DAX can be used to generate dynamic insights from transactional datasets.

---

## 4. Report and Dashboard Creation

Created interactive reports and dashboards to visualize business performance.

The report includes:

- Sales analysis visuals
- Product performance insights
- Customer-based reporting
- Regional sales breakdown
- Return analysis
- KPI cards and comparative metrics
- Time-based trend analysis

Interactive filtering and slicing were also used to improve report usability and user experience.
